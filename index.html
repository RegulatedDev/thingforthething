Protocols
Rules that define how data is transmitted over the internet. HTTP and HTTPS are the main protocols for web browsing, ensuring your browser and servers speak the same language.
Routers
Network devices that direct your data packets to their destination. Like a postal sorting facility, routers determine the best path for your request to travel across the internet.
IP Addresses
Unique numerical identifiers for every device on the internet (like 192.168.1.1). Think of it as the street address for computers, allowing them to find and communicate with each other.
Servers
Powerful computers that store and deliver web pages, images, and data. When you visit a website, you're requesting files from a server somewhere in the world.
Domain Names
Human-readable addresses like "google.com" or "wikipedia.org". They're easier to remember than IP addresses and get translated by DNS into actual server locations.
Packets
Small chunks of data that your request is broken into for transmission. Like breaking a letter into sentences, packets travel independently and reassemble at their destination.
Web Designer
The person who plans how a website looks and feels. They focus on visual layout, colors, typography, and user experience before development begins.
Host
The service or computer that stores your website files and makes them available on the internet 24/7. Web hosting companies provide servers where websites live.
UI (User Interface)
All the visual elements you interact with on a webpage: buttons, menus, forms, images. It's everything you see and click on when using a website.
UX (User Experience)
How users feel when interacting with a website. Good UX means the site is intuitive, fast, accessible, and enjoyable to use from start to finish.
HTML
HyperText Markup Language - the backbone structure of web pages. HTML defines the content and organization: headings, paragraphs, links, images, and more.
CSS
Cascading Style Sheets - the styling language that makes HTML look good. CSS controls colors, fonts, spacing, layouts, and animations on web pages.
Web Developer
The person who writes the code to build websites. They turn designs into functional pages using HTML, CSS, JavaScript, and other technologies.
Request
When your browser asks a server for a web page or resource. Every time you click a link or type a URL, your browser sends an HTTP request.
Render
The process of your browser converting HTML, CSS, and JavaScript into the visual page you see. Rendering transforms code into pixels on your screen.
Respond
When a server sends back the requested data to your browser. The response includes the HTML, CSS, images, and other files needed to display the page.
Internet
The global network of interconnected computers and servers. It's the infrastructure that allows devices worldwide to communicate and share information.
URL
Uniform Resource Locator - the complete web address like "https://example.com/page.html". It tells your browser exactly which protocol, domain, and resource to access.
Path
The specific location of a resource on a server, like "/images/photo.jpg". It comes after the domain name in a URL and points to a particular file or page.
Resources
Any files that make up a web page: HTML documents, CSS stylesheets, JavaScript files, images, videos, and fonts. Browsers request all necessary resources to build the complete page.
Port
A numbered channel on a server that handles specific types of traffic. Web traffic typically uses port 80 (HTTP) or 443 (HTTPS), like different doors to the same building.
Interpret
How your browser reads and executes code. The browser interprets HTML to understand structure, CSS for styling, and JavaScript for interactive behavior.
DNS
Domain Name System - the internet's phonebook. DNS translates human-friendly domain names (like "google.com") into IP addresses that computers use to locate servers.
End-user
You! The person who visits and uses websites. Everything in web development is ultimately designed with the end-user's needs and experience in mind.
